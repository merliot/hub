TEMPLATE FUNCTIONS

Each device model includes a set of built-in template helper functions, making it easier to include device state in template output. For example, use functions target and state to return a custom html snippet:

<div>
	{{ if eq target "rpi" }}
		<span>Raspberry Pi {{state.Foo}}</span>
	{{ else }}
		<span>Not Raspberry Pi {{state.Bar}}</span>
	{{ end }}
</div>
Function
Description
id() string
Returns device ID
model() string
Returns device model name
name() string
Return device name
uniq(s string) string
Returns unique id := model() + id() + s
deployParams() template.HTML
Returns device deploy params escaped as template.HTML
state() any
Returns device state (see hub.Config.State). Only exported fields in the device state type are visible. For example, {{state.Foo}} is available, but {{state.foo}} isn't.
title(s string) string
Returns string.Title(s)
add(a, b int) int
Returns a + b
mult(a, b int) int
Returns a * b
joinStrings(parts ...string) string
Returns strings.Join(parts, "")
target() string
Returns device target
