MCP SERVER

Merliot Hub supports Model Context Protocol (MCP). MCP allows LLM hosts, such as Claude Desktop, to "plug-in" to external context. Merliot Hub includes a MCP Server which lets LLM hosts "plug-in" to the hub. See my blog post.

Now, using the LLM, we can "chat" with the hub using natural language:

"List all of the devices in a tree view"
"Add a new gps device"
"Turn on all the relays"
"Show the instructions on how to deploy a qrcode device"

INSTALL

Click on the tool icon on the hub:

Follow the instructions to download and install the MCP server for your local platform. The MCP server only supports stdio transport. The MCP server will automatically connect to the hub once plugged into an LLM host. More than one LLM host may be using a MCP Server to connect to the hub.

TOOLS

Here's a list of MCP tools available to the LLM:

Tool Name Description
add_device Add a new device to the Merliot Hub
camera_get-image Get the camera image
gadget_takeone Take one down, pass it around
get_config Get the model configuration of a Merliot Hub device model
get_devices Get all devices running on the Merliot Hub
get_instructions Get the instructions for device on the Merliot Hub
get_models Get list of all device models available on the Merliot Hub
get_params Get the parameter schema for a Merliot Hub device model
get_server_status Get the status of the Merliot Hub
get_state Get the state of a device on the Merliot Hub
get_status Get the status of a Merliot Hub device
qrcode_update QR code
relays_click Click (toggle) the relay
remove_device Remove a device from the Merliot Hub
rename Rename a device on the Merliot Hub
save Save current device configuration
